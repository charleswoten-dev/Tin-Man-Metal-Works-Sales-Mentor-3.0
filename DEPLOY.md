# Deploying Tin Man Sales Mentor

The app has two deployables plus a managed database:

| Piece    | What it is                         | Host                  |
| -------- | ---------------------------------- | --------------------- |
| Client   | React PWA (`client/`), static build| **Vercel**            |
| Server   | Express API (`server/`), Node      | **Render**            |
| Database | Supabase (auth + Postgres + RLS)   | Already live (managed)|

Deploy the **server first** — you need its public URL to configure the client.

---

## 1. Server → Render

The server holds every secret. The service-role key and API keys live ONLY here,
never in the client bundle or in git.

1. In Render: **New → Blueprint**, point it at this GitHub repo. Render reads
   `render.yaml` and creates the `tin-man-server` web service (rootDir `server`,
   `npm install` / `npm start`, health check `/api/health`).
2. Fill in the secrets Render prompts for (everything marked `sync: false`):

   | Env var                       | Value                                                        |
   | ----------------------------- | ------------------------------------------------------------ |
   | `ANTHROPIC_API_KEY`           | From console.anthropic.com → API Keys                        |
   | `ANTHROPIC_MODEL`             | `claude-sonnet-4-6` (preset)                                 |
   | `SUPABASE_URL`                | `https://retufxuvstfupekimvhk.supabase.co`                   |
   | `SUPABASE_SERVICE_ROLE_KEY`   | Supabase → Project Settings → API → service_role (SECRET)    |
   | `CLICKFUNNELS_WEBHOOK_SECRET` | A strong random string (reuse it in ClickFunnels, step 3)    |
   | `RESEND_API_KEY`              | From resend.com → API Keys                                   |
   | `EMAIL_SERVICE`               | `resend` (preset)                                            |
   | `EMAIL_FROM`                  | e.g. `Tin Man <noreply@tinmanmetalworks.com>`                |
   | `APP_URL`                     | The live client URL (fill in after step 2, e.g. the Vercel domain) |
   | `ADDITIONAL_ORIGINS`          | Optional — extra allowed origins, comma-separated            |
   | `ADMIN_JWT_SECRET`            | Auto-generated by Render                                     |
   | `ADMIN_EMAIL` / `ADMIN_PASSWORD` | Strong, unique admin login for `/admin`                   |

3. Deploy. Confirm `https://<your-render-app>.onrender.com/api/health` returns OK.
   Note this URL — it's the client's `VITE_API_BASE`.

> Note: Render's **free** plan sleeps after inactivity (first request after a nap
> is slow). Use the **starter** plan for production so the API stays warm.

---

## 2. Client → Vercel

1. In Vercel: **Add New → Project**, import this repo.
2. Set **Root Directory = `client`** (the rest is handled by `client/vercel.json`:
   Vite framework, `npm run build`, output `dist`, SPA rewrites so deep links work).
3. Add **build-time** environment variables (Project → Settings → Environment Variables):

   | Env var                  | Value                                          |
   | ------------------------ | ---------------------------------------------- |
   | `VITE_SUPABASE_URL`      | `https://retufxuvstfupekimvhk.supabase.co`     |
   | `VITE_SUPABASE_ANON_KEY` | Supabase anon key (safe for the browser)       |
   | `VITE_API_BASE`          | The Render server URL from step 1 (no trailing `/`) |

   > These are PUBLIC (bundled into the browser). Never put the service-role key
   > or the Anthropic key here.

4. Deploy. Copy the resulting URL (e.g. `https://tin-man.vercel.app`).
5. **Go back to Render** and set `APP_URL` to that exact URL (this is what the
   CORS allowlist checks), then redeploy the server so the client can call it.

---

## 3. ClickFunnels purchase webhook

So a purchase mints a license key and emails it to the buyer:

1. In ClickFunnels, add a post-purchase webhook pointing to:
   `https://<your-render-app>.onrender.com/api/webhooks/clickfunnels`
2. Configure it to sign requests with the **same** secret you set as
   `CLICKFUNNELS_WEBHOOK_SECRET`. The server rejects any unsigned/mismatched
   request (401), so these must match exactly.

---

## 4. Email deliverability (Resend)

In Resend, **verify your sending domain** (add the DNS records they provide).
Until the domain is verified, welcome emails with the license key will land in
spam or fail. `EMAIL_FROM` must use the verified domain.

---

## 5. Custom domain (optional but recommended)

- Point e.g. `app.tinmanmetalworks.com` at the Vercel project.
- Update `APP_URL` on Render to the custom domain and redeploy.
- If both the Vercel `*.vercel.app` URL and the custom domain need to work, put
  the secondary one in `ADDITIONAL_ORIGINS`.

---

## 6. Final go-live smoke test

Do one real end-to-end run before announcing:

1. Make a test purchase in ClickFunnels.
2. Confirm the webhook succeeded (Render logs, or the `/admin` panel).
3. Confirm the welcome email with the `TM3-XXXX-XXXX-XXXX` key arrives.
4. Register with that email + key → set a password → log in.
5. Send a chat message and run a couple of walkthrough steps.
6. Try the key a second time — it must be rejected (single-use).
7. Log into `/admin` with your admin credentials.

If all seven pass, you're live.

---

## Env var quick reference

**Server (Render — secrets, never in git):** `ANTHROPIC_API_KEY`,
`ANTHROPIC_MODEL`, `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`,
`CLICKFUNNELS_WEBHOOK_SECRET`, `RESEND_API_KEY`, `EMAIL_SERVICE`, `EMAIL_FROM`,
`APP_URL`, `ADDITIONAL_ORIGINS` (optional), `ADMIN_JWT_SECRET`, `ADMIN_EMAIL`,
`ADMIN_PASSWORD`, `PORT` (Render injects automatically).

**Client (Vercel — public, build-time):** `VITE_SUPABASE_URL`,
`VITE_SUPABASE_ANON_KEY`, `VITE_API_BASE`.
